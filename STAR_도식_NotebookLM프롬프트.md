## 도식 1 — 라스트100m '측위 공백' 개념도  (삽입 위치: 1-2 개발목적/해결방안 또는 2-2 차별화)
**프롬프트:**
"보행자 길찾기 'Last 100m' 문제를 위성항법(GNSS) 측위 관점에서 설명하는 1장짜리 인포그래픽 슬라이드를
한국어로 만들어줘. 왼쪽: 개활지에서 위성 신호가 곧게 들어와 측위가 정확한 모습. 오른쪽: 도심 협곡(고층빌딩
사이)·좁은 골목에서 위성 신호가 건물에 반사(멀티패스)·차폐되어 위치 점이 실제 위치에서 벗어나 흩어지는
모습(오차 영역을 원으로 표시). 하단에 '라스트 100m 실패의 물리적 근본원인 = 위성항법이 무력화되는 마지막
구간'이라는 한 줄 메시지. 색은 남색 계열, 깔끔한 정부제출용 톤."

## 도식 2 — 항공우주 2축 역할분리도  (삽입 위치: 2-1 개발방안 / 진행정도)
**프롬프트:**
"도보네비게이션이 항공우주 기술을 활용하는 방식을 '두 축'으로 분리해 보여주는 다이어그램 슬라이드를
한국어로. 위쪽 축: '위성항법(GNSS·KPS 연계) = 실시간 정밀측위'(아이콘: 위성→사람 위치). 아래 축:
'위성영상 = 보행공간 데이터 생산(건물 출입구·골목 동선·랜드마크·계단/언덕을 추출·3D화)'(아이콘: 위성→
지도 레이어). 두 축이 가운데 'AI 랜드마크 길안내 엔진'으로 합류해 화면(보행 경로)으로 출력되는 흐름.
주석: 'KPS는 향후 연계·실증 대상, 현재 기반은 GPS/GNSS·SBAS'. 정부제출용 남색 톤."

## 도식 3 — 서비스 동작 흐름도  (삽입 위치: 2-1 또는 1-2)
**프롬프트:**
"보행자 도보 내비게이션 서비스의 동작 흐름을 좌→우 5단계로 보여주는 플로우 슬라이드(한국어):
①위성영상으로 출입구·골목·랜드마크 데이터화 → ②위성항법(GNSS) 측위 + 음영구간 보정 → ③AI가 '큰 약국
끼고 좌회전'식 랜드마크 기반 경로 설명 생성 → ④사용자에게 도보 경로 안내 → ⑤경로 이탈 시 재안내 알림.
각 단계 아이콘과 한 줄 설명. 깔끔한 남색 톤, 정부제출용."

## 도식 4 — 멘토링 4개월 추진 일정(간트)  (삽입 위치: 3-2 사업 추진 일정)
**프롬프트:**
"2026년 8~11월 멘토링 4개월 추진 일정을 간트차트 형태 슬라이드로(한국어). 4개 막대: 1단계(8월)
사용자 인터뷰·현장 동선조사 / 2단계(9월) 지도 API 기반 경로안내 MVP / 3단계(10월) AI 경로추천·로컬
관광코스 고도화 / 4단계(11월) 소상공인·관광사업자 B2B 검증. 하단 마일스톤 배지: '보행 불편 데이터
100건↑ · 테스트 경로 10개 · 초기 사용자 100명 검증' + '위성항법 보정이 필요한 보행 구간 규명'. 남색 톤."

## 도식 5 — 시장 규모 TAM/SAM/SOM  (삽입 위치: 1-3 목표시장 분석)
**프롬프트:**
"동심원(또는 깔때기) 형태로 TAM-SAM-SOM을 보여주는 시장규모 슬라이드(한국어). TAM 약 1.0조 원(전국
유동의존 점포 약 290만개 × 연 36만원) → SAM 약 1,600억 원(서울·주요 관광/골목상권 약 45만개) → SOM
약 18억 원(홍대·연남·합정 약 5,000개, 침투율 10% 가정). ARPU 월 3만원 표기. 출처·집계 방식은 사업화
단계 보강 예정이라는 작은 주석. 정부제출용 남색 톤."
