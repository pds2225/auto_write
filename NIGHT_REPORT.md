### 표 셀 안 인라인 빈칸(`성명 : ____   생년월일 : ____`)을 이제 자동 전사합니다

- **전에는:** 완성본 A → 빈 양식 B 자동 채우기(cross-form)가 **표의 "라벨 칸 +
  옆 값 칸"** 과 **본문 줄** 두 가지만 채웠습니다. 그런데 정부양식 표지·개요 박스는
  한 칸 안에 `신청기업명 : ____________    대표자 : ______` 처럼 **여러 항목을
  나란히** 넣는 경우가 아주 많습니다. 이런 "한 칸 안 인라인 빈칸"은
  **전혀 인식되지 않아 하나도 안 채워졌습니다**(프로그램이 표 셀 안쪽 글줄을
  못 들여다봤기 때문). 사용자가 그 칸들을 일일이 손으로 옮겨 적어야 했습니다.
- **이제는:** 표 셀 안의 `라벨 : ______` 형태 빈칸을 **직접 찾아 값을 채웁니다.**
  한 칸에 여러 항목이 있어도 각각 채우고(예: `신청기업명`·`대표자` 동시),
  **라벨·콜론·칸 간격은 그대로 두고 밑줄 빈칸 자리에만** 값을 넣습니다.
  동의어(예: 양식의 `성명` ← 완성본의 `대표자`)도 그대로 인식합니다.
- **안전(불변 원칙 그대로):**
  · **밑줄·점 같은 "보이는 빈칸"만** 채웁니다. `라벨 :`(뒤가 그냥 빈) 형태는 옆 칸을
    가리키는 경우와 헷갈릴 수 있어 **건드리지 않습니다**(엉뚱한 자리 기입 방지).
  · 이미 값(마스킹 `○○○` 포함)이 든 칸은 **덮어쓰지 않습니다.**
  · 완성본에 값이 없는 항목은 **빈칸으로 그대로 둡니다**(없는 값 지어내기 0).
  · 소스·타깃 **원본 파일은 절대 수정하지 않습니다**(결과는 새 파일로만 저장).

### 제출 진단이 "이 문서의 실제 빈 칸 이름"을 알려 주고, 체크리스트 파일로 저장됩니다

  화면 위쪽 기술 목록을 뒤져야 알 수 있어, 비개발자에겐 "뭘 채우라는 거지?"가
  남았습니다.
- **이제는 세 가지가 새로 됩니다.**
  1. **콕 집어 알려 줍니다** — 각 결함 밑에 **그 문서의 실제 위치**를 나열합니다.

  2. **얼마나 남았는지 한 줄로** — 맨 위에 **"제출까지 남은 일 요약: 자동 명령 2개 실행

     "제출까지 얼마 남았나"가 한눈에 들어옵니다.
  3. **체크리스트 파일로 저장** — `--checklist 제출준비.md` 를 붙이면, 남은 일을
     **체크박스 to-do 문서**(`- [ ] 대표자명 옆 칸 …`)로 저장합니다. 열어서 하나씩
     `- [x]` 로 지워 가며 제출 준비를 완성할 수 있고, 복사해서 실행할 자동 명령
     묶음도 파일 끝에 정리됩니다.
- **기계도 그대로 씁니다:** `--json` 결과에 각 항목의 실제 위치(`items`)와
  진행도 요약(`remediation_summary`)이 함께 담겨, autopilot·에이전트가 "채울 칸
  목록"을 바로 이어받을 수 있습니다.
- **안전:** 진단은 **읽기 전용**이라 원본을 절대 건드리지 않습니다. 체크리스트 저장

  값이 없는 칸은 "사람이 입력"으로 분류하고 **없는 값을 지어내라고 하지 않습니다**(날조 0).
