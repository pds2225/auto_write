# announcement-form-analysis — 정부지원사업 공고·양식 통합 분석

> 완성된 분석 결과물은 "지원자가 바로 판단·작성에 쓸 수 있는 표"다.
> 단순 텍스트 받아쓰기가 아니라 **조건 정리 + 양식 비교 + 가성비 추천**까지가 산출물.

## 목적

한 폴더에 흩어진 **공고문 1개 + 운영기관별 사업계획서 양식 N개**(보통 HWP)를 받아:
1. 공고의 핵심 조건·마감·제출서류·제외대상·평가일정을 정리하고,
2. 양식 N개를 **비교표**로 만들어 분량·평가성격·작성항목 차이를 드러내고,
3. 신청자 상황에 맞춰 **어느 운영기관 1곳에 지원할지 + 무엇을 강조해 쓸지**를 추천한다.

## 언제 사용하나

- "이 폴더에 있는 공고문과 양식 분석해줘"
- "공고문만 정리해줘" / "양식만 비교해줘" (부분 분석)
- "양식 하나 더 추가됐는데 같이 비교해줘" (재실행·증분)
- 정부지원사업/창업지원사업/액셀러레이팅/바우처 공고·양식 검토 일반

## 입력

- 분석 대상 폴더 경로 (예: OneDrive·카카오톡 받은 파일 등 외부 폴더도 가능)
- 파일 형식: `.hwp`, `.hwpx`, `.docx` (혼재 가능)
- (선택) 신청자 상황: 지역(수도권/비수도권), 업종, 매출·투자·특허 실적, 예비 or 기창업 — 있으면 추천 정확도↑

## 워크플로우

### Step 0. 폴더 스캔
- `Get-ChildItem -LiteralPath "<폴더>" -Force` 로 파일 목록·크기 확인.
- **한글 파일명/OneDrive 폴더는 NFC/NFD 정규화 차이로 Python이 경로 문자열로는 못 여는 경우가 많다.** 절대 `python ... "한글경로"` 로 직접 열지 말 것. 반드시 Step 1의 `Path.glob()` 으로 잡는다.
- 어떤 게 공고문이고 어떤 게 양식인지 파일명으로 1차 분류(`[공고문]`, `모집공고` = 공고 / `사업계획서 양식_OOO` = 운영기관별 양식).

### Step 1. HWP/HWPX/DOCX 텍스트 추출 (auto_write 함수 재사용)
- **HWP는 직접 못 읽는다.** `hwp5`/`pyhwp` 불필요(미설치). auto_write 프로젝트의 `olefile`(설치됨) 기반 추출기를 그대로 쓴다.
- 추출 스크립트를 job tmp(또는 `app/tmp_quality_input/` 같은 ASCII 경로)에 만들어 실행:

```python
# -*- coding: utf-8 -*-
import sys
from pathlib import Path
sys.path.insert(0, r"D:\auto_write\app")           # import 기준
from auto_write.document_ingest import extract_additional_text

SRC = Path(r"<분석 대상 폴더>")
OUT = Path(r"<ASCII 출력 폴더>"); OUT.mkdir(parents=True, exist_ok=True)
for i, f in enumerate(sorted(SRC.glob("*.hwp"))):   # glob 으로 한글경로 우회
    text = extract_additional_text(f)               # .hwp/.hwpx 만. (.docx 는 python-docx 로 별도)
    (OUT / f"{i:02d}_{f.stem}.txt").write_text(text or "", encoding="utf-8")
    print(f"[{i:02d}] {f.name} chars={len(text or '')}")
```

- 실행: `$py = "C:\Users\ekth3\AppData\Local\Programs\Python\Python311\python.exe"; & $py <script>`
- **콘솔에서 한글이 깨져 보여도 무시** — 파일은 UTF-8로 정상 저장된다. 추출 결과는 `Read` 툴로 읽으면 정상.
- `extract_additional_text` 내부: `.hwp` → `extract_hwp_markdown_text` → 실패 시 `extract_hwp_preview_text(PrvText 스트림)` 폴백. 표는 markdown 표로 어느 정도 복원됨.

### Step 2. 공고문 분석 (1개)
추출 텍스트에서 아래 9요소 + 마감 경고를 표로 정리:
- 사업명/주관기관, 지원대상(자격), 지원금·자기부담 구조, 선정규모, **신청방법(중복신청 가능 여부)**, 제출서류, 평가일정(접수→서류→발표→결과), **신청제외 대상**, 가점.
- **⚠️ 마감일 경고(필수):** 공고의 접수 마감일을 `currentDate`(오늘)와 비교해, **오늘이 마감이거나 임박/지났으면 답변 맨 앞에 굵게 경고**한다. (실제 사례에서 가장 중요했던 인사이트 — 오늘이 마감인 줄 모르면 분석이 무의미)

### Step 3. 양식 비교 (N개)
운영기관별 양식을 **가로 비교표**로:
- 지역 제한(전국/비수도권 전용 등), 직접투자·지원 규모, 양식 분량(쪽수), 평가 성격(정성·스토리 / 정량·실적 / 트랙적합 등), 평가지표 배점, 핵심 작성항목·특이항목(예: 특정 대학 인프라 활용계획, 트랙 선택).
- 공고문 [참고]의 **운영기관별 선정기준**과 각 양식의 **작성항목**을 교차 매핑(평가에서 보는 것 ↔ 양식에서 쓰라는 것).

### Step 4. 가성비 추천
신청자 상황(없으면 일반 시나리오)별로 "어디 1곳 + 무엇을 강조":
- 실적 약한 초기/예비 → 정성·스토리 평가 기관 또는 양식 가벼운 기관.
- 매출·투자·IP 실적 강함 → 정량 비중 큰 기관.
- 투자유치 목표 → 직접투자 규모 최대 기관.
- 시간 없음 → 양식 최경량 기관(단, 지역 제한 확인).

### Step 5. 후속 연계 제안
- 신청자 상황을 받아 **운영기관 1곳 확정** → 그 양식으로 **사업계획서 초안 자동작성**(`auto-write-bizplan` 오토파일럿, HWP→DOCX 변환 후) 안내.

## 핵심 제약·함정 (검증됨)

- **HWP 직접 읽기 금지** → 반드시 `extract_additional_text`. `Read` 툴로 .hwp 직접 열면 깨진 바이너리.
- **한글/OneDrive 경로** → `Path.glob("*.hwp")`로 잡고 ASCII 출력 폴더에 UTF-8 저장. 경로 문자열 직접 전달 금지.
- **콘솔 한글 깨짐 = 정상**(cp949 출력 한계). 파일 인코딩(UTF-8)과 무관.
- **추출 품질 차이**: 양식이 표 위주면 셀이 줄바꿈으로 풀려 나올 수 있음(예: 짧은 양식은 840자처럼 나옴 = 원래 짧은 양식). 너무 짧으면 `extract_hwp_preview_text` 폴백 결과인지, 원래 짧은 양식인지 원문 길이로 구분.
- **숫자/조건 날조 절대 금지**: 공고에 없는 마감일·배점·금액을 지어내지 않는다. 배점이 공고에 없으면 "공고에 명시 없음"으로 표기(균등 배점 가정 시 가정임을 명시).
- **1개 운영기관만 선택형 공고** 주의: 중복신청 불가 조건을 추천에서 반드시 반영.
- 분석 전용(읽기). 원본 HWP를 수정·덮어쓰지 않는다.

## 성공 기준

- [ ] 폴더 내 모든 대상 파일이 추출됨(파일 수 = 추출 수, chars>0).
- [ ] 공고 9요소 + **마감일 경고**가 답변에 포함됨.
- [ ] 양식 N개가 1개 가로 비교표로 정리됨(지역/규모/분량/평가성격/배점/특이항목).
- [ ] 신청자 상황(또는 일반 시나리오)별 운영기관 추천이 제시됨.
- [ ] `result:` 한 줄 헤드라인으로 핵심(마감·기관 수·최대투자·최경량 양식 등) 요약.

## 검증

- 추출 직후 각 파일 `chars`/`lines` 출력으로 0바이트·실패 즉시 확인.
- 의심되면 추출 .txt를 `Read`로 열어 한글 정상 여부 육안 확인.
- 공고에서 뽑은 마감일·배점·금액은 원문 줄을 근거로 인용(날조 방지).

## 환경 (검증된 사실)

- 실행 Python: `C:\Users\ekth3\AppData\Local\Programs\Python\Python311\python.exe` (venv 없음).
- import 기준: `D:\auto_write\app` (`from auto_write.document_ingest import ...`).
- 필요한 라이브러리: `olefile`(설치됨), `win32com`(설치됨, 미사용). `hwp5`/`pyhwp` 불필요.

## 열린 항목 (아직 자동화 못 한 부분)

- 표가 복잡한 양식은 셀 경계가 풀려 나와, 비교표 작성 시 사람이 항목을 재구성해야 할 수 있음.
- 운영기관별 배점이 공고·양식에 숫자로 없으면 정량 비교 불가(가정 명시 필요).
- HWP→DOCX 변환(후속 자동작성 연계)은 `unhwp` 가용 시에만 표 구조까지 복원됨(없으면 미리보기 텍스트 기반).
