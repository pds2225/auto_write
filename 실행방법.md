# Auto Write 실행방법

## 1. 결론
`D:\auto_write`는 사업계획서 작성시스템입니다. 처음 사용하는 경우 아래 순서대로 실행합니다.

```text
1. setup.bat 실행
2. check_env.bat 실행
3. launch.bat 실행
4. 브라우저에서 http://127.0.0.1:8765 접속
```

---

## 2. 각 파일의 역할

| 파일 | 역할 | 언제 실행하나 |
|---|---|---|
| `setup.bat` | 필요한 Python 패키지를 설치 | 최초 1회 또는 오류 발생 시 |
| `check_env.bat` | Python, 패키지, 앱 import 상태 점검 | 설치 후 또는 실행 오류 발생 시 |
| `launch.bat` | Auto Write 웹앱 실행 | 실제 사용할 때마다 |

---

## 3. 최초 실행 순서

### 1단계. 설치

Windows 파일 탐색기에서 아래 파일을 더블클릭합니다.

```text
D:\auto_write\setup.bat
```

설치가 끝나면 창에 `설치 완료`가 표시됩니다.

### 2단계. 실행환경 점검

아래 파일을 더블클릭합니다.

```text
D:\auto_write\check_env.bat
```

정상이라면 다음과 같은 메시지가 표시됩니다.

```text
[OK] 핵심 패키지 import 성공
[OK] auto_write.main import 성공
점검 완료: launch.bat 실행 가능 상태로 보입니다.
```

### 3단계. Auto Write 실행

아래 파일을 더블클릭합니다.

```text
D:\auto_write\launch.bat
```

잠시 후 브라우저가 열리며 아래 주소로 접속됩니다.

```text
http://127.0.0.1:8765
```

브라우저가 자동으로 열리지 않으면 주소창에 위 주소를 직접 입력합니다.

---

## 4. 기본 사용 흐름

| 순서 | 작업 | 설명 |
|---|---|---|
| 1 | 템플릿 업로드 | 사업계획서 DOCX/HWP/HWPX 양식 업로드 |
| 2 | 템플릿 분석 | 문단, 표 입력칸, 이미지 위치 자동 분석 |
| 3 | 프로젝트 생성 | 분석된 템플릿으로 작성 프로젝트 생성 |
| 4 | 핵심 정보 입력 | 과제명, 회사명, 사업 개요, 추가 메모 입력 |
| 5 | 참고자료 업로드 | 공고문, 상담메모, 기존 사업계획서 등 업로드 |
| 6 | 생성 실행 | 사업계획서 초안 및 DOCX 생성 |
| 7 | 결과 확인 | DOCX, QA 리포트, 출처 목록 다운로드 |

---

## 5. API 키 설정

AI 문안 생성 품질을 높이려면 API 키가 필요합니다.

### 설정 파일

```text
D:\auto_write\app\.env
```

`D:\auto_write\app\.env.example` 파일을 복사해서 `.env`로 이름을 바꾼 뒤 아래 중 하나를 입력합니다.

```text
OPENAI_API_KEY=발급받은_OpenAI_키
```

또는

```text
ANTHROPIC_API_KEY=발급받은_Anthropic_키
```

> 주의: API 키는 외부에 공유하지 않습니다.

## 6. 오류 발생 시 조치

| 증상 | 조치 |
|---|---|
| Python을 찾지 못함 | Python 3.11 이상 설치 후 다시 실행 |
| 패키지 import 실패 | `setup.bat` 재실행 |
| 웹페이지 접속 안 됨 | `launch.bat` 실행창의 오류 메시지 확인 |
| `http://127.0.0.1:8765` 접속 실패 | 다른 프로그램이 포트를 사용 중인지 확인 |
| 템플릿 분석 실패 | DOCX 파일로 변환 후 재시도 |
| AI 문안 품질이 낮음 | `.env`에 API 키 설정 여부 확인 |

오류 메시지가 나오면 창의 내용을 복사해서 AI에게 전달하면 됩니다.

## 7. 현재 MVP 목표

`auto_write`의 1차 완성 목표는 다음입니다.

| 기능 | 완료 기준 |
|---|---|
| 실행 | `launch.bat` 실행 시 웹 화면이 열린다 |
| 템플릿 업로드 | 사업계획서 양식 업로드 후 분석 결과가 표시된다 |
| 프로젝트 생성 | 템플릿 기반 프로젝트를 만들 수 있다 |
| PSST 초안 생성 | Problem-Solution-Scale-Team 관점의 사업계획서 문안이 생성된다 |
| DOCX 출력 | 생성 결과가 `output.docx`로 저장된다 |
| QA 점검 | 누락항목, 오류, 경고가 리포트로 표시된다 |
| 다운로드 | 결과 DOCX와 리포트를 받을 수 있다 |

---

## 8. 작업 원칙

- 기존 파일은 삭제하지 않습니다.
- 기존 파일 수정 전 백업본을 만듭니다.
- 대량 이동 또는 삭제 전에는 사용자 승인을 받습니다.
- 오류 원인을 추정하지 않고, 확인된 메시지 기준으로 처리합니다.
